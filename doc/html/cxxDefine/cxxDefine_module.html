cxxDefine module

Defines a top-level container for all the elements necessary to describe a preprocessor definition.

Description

The <cxxDefine> topic type contains elements to show the name of the preprocessor definition, the #define syntax, and a description of the use of the #define.

The cxxDefine topic contains the following elements:

  • cxxDefine
    The <cxxDefine> element provides child elements describing a C++ preprocessor #define.
  • cxxDefineAPIItemLocation
    Specifies the location of a preprocessor symbol introduced using #define.
  • cxxDefineDeclarationFile
    Specifies the file containing the #define.
  • cxxDefineDeclarationFileLine
    Specifies the line number containing the #define.
  • cxxDefineDefinition
    Contains child elements that provide a description of a preprocessor definition introduced through the #define directive.
  • cxxDefineDetail
    The <cxxDefineDetail> element contain child elements to fully describe a #define preprocessor directive.
  • cxxDefineNameLookup
    Provides a unique identifier of a preprocessor #define directive.
  • cxxDefineParameter
    When a #define is used as preprocessor macro, the <cxxDefineParameter> element describes a named macro parameter.
  • cxxDefineParameterDeclarationName
    The <cxxDefineParameterDeclarationName> element specifies the name of a parameter used by a preprocessor macro.
  • cxxDefineParameters
    A #define that takes arguments (or parameters) is commonly called a macro. This element allows you to describe the purpose of the identifier list that makes up the macro arguments.
  • cxxDefinePrototype
    Gives a view of a preprocessor #define directive as it looks in the source code.
